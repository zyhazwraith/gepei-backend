# FP-005 地陪展示与搜索 (Guide Discovery) 开发计划

根据你的要求，我们将按照 **后端 -> 自测 -> 前端 -> 自测 -> 转测报告** 的流程进行开发。

## 1. 后端开发 (Backend)
- **目标**: 提供公开的地陪列表查询接口，支持筛选。
- **任务**:
    1.  **Model 层**: 在 `server/models/guide.model.ts` 中添加 `findAllGuides` 方法，支持分页、城市过滤、关键词搜索。
    2.  **Controller 层**: 在 `server/controllers/guide.controller.ts` 中实现 `getGuides` 方法。
    3.  **Route 层**: 在 `server/routes/guide.routes.ts` 中添加 `GET /` 路由（注意：此接口应为公开接口，不需要 `authenticate` 中间件，或者可选认证）。
- **自测**: 使用 `curl` 或编写简单脚本验证接口返回数据的正确性。

## 2. 前端开发 (Frontend)
- **目标**: 展示地陪列表，对接真实数据。
- **任务**:
    1.  **API 封装**: 在 `client/src/lib/api.ts` 中添加 `getGuides` 方法。
    2.  **列表页**: 修改 `client/src/pages/Guides.tsx`，使用真实 API 替换 Mock 数据，实现搜索和筛选交互。
    3.  **首页**: 修改 `client/src/pages/Home.tsx`，让"热门地陪"部分加载真实数据（取前4-8条）。
- **自测**: 在浏览器中操作，验证搜索、翻页和加载状态。

## 3. 交付 (Handover)
- **任务**:
    1.  编写转测报告 `docs/qa_handovers/FP005_HANDOVER.md`。
    2.  提交代码到 `feature/FP-005-guide-discovery`。
