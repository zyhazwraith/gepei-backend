# 冒烟测试计划

环境已就绪，但数据库可能还是空的（Setup 脚本只创建了数据库，未创建表）。为了确保系统正常运行，我建议按以下步骤进行冒烟测试：

## 1. 数据库初始化
- **操作**: 运行 `npx drizzle-kit push`
- **目的**: 将 `server/db/schema.ts` 中定义的表结构同步到 MySQL 数据库中。

## 2. 启动服务
- **操作**: 运行 `npm run dev` (同时启动后端和前端)
- **目的**: 确保后端 API 服务 (Port 3000) 和 前端页面 (Port 5173) 能正常启动。

## 3. API 冒烟测试 (后端)
- **健康检查**: 访问 `GET /health`，确认返回 `Server is running`。
- **用户注册**: 测试 `POST /api/v1/auth/register`，创建一个测试用户。
- **用户登录**: 测试 `POST /api/v1/auth/login`，确认能获取 Token。

## 4. 页面冒烟测试 (前端)
- **操作**: 提供预览链接，让你在浏览器中查看首页。
- **验证**: 确认页面加载无报错，且能看到基本的 UI。

如果确认，我将开始执行这些步骤。
