# 全量测试数据生成计划

好的，我已仔细审查了 `schema.ts` 中的所有 9 张表。除了 `admin_logs` 之外，其他表都与核心业务流程紧密相关。

为了支持后续的开发（如订单、支付、评价等模块），我建议生成覆盖这些表的 Mock 数据。

## 1. 覆盖范围 (Schema Coverage)
| 表名 | 生成策略 | 数量 |
| :--- | :--- | :--- |
| **users** | 随机手机号、头像、昵称 | 50 (含1个固定测试账号 `13800138000`) |
| **guides** | 从 Users 中随机选取，覆盖多城市、价格区间 | 30 |
| **orders** | 生成各种状态 (pending/paid/completed) 的订单 | 50+ |
| **payments** | 对应已支付订单的流水 | 30+ |
| **reviews** | 对应已完成订单的评价 | 20+ |
| **withdrawals** | 地陪的提现记录 | 10 |
| **custom_requirements** | 定制需求单 | 5 |
| **custom_order_candidates** | 定制单的候选地陪 | 10 |
| **admin_logs** | **跳过** (无需 Mock) | 0 |

## 2. 技术实现
- **库**: 使用 `@faker-js/faker` 生成逼真数据。
- **脚本**: `scripts/seed-data.ts`。
- **逻辑**: 
    1. 清理数据（可选）。
    2. 创建用户。
    3. 升级部分用户为地陪。
    4. 创建普通用户与地陪之间的订单。
    5. 为订单生成支付、评价记录。

## 3. 执行步骤
1.  安装依赖: `npm install @faker-js/faker --save-dev`。
2.  编写并运行脚本。
3.  验证数据。
