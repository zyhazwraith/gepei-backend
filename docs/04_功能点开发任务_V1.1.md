**API规范**:
- 端点: POST /api/v1/auth/register
- 请求: { phone, password, nickname }

**验证方式**:
```bash

  -H "Content-Type: application/json" \
  -d '{"phone":"13800000000","password":"password123","nickname":"张三"}'

# 应该返回 { code: 0, data: { user_id, token, ... } }

# 重复注册应该返回错误
curl -X POST http://localhost:3000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800000000","password":"password123","nickname":"李四"}'

# 应该返回 { code: 1001, message: "手机号已注册" }
```

## 代码提交规范

### Commit Message格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type** (必须):
- feat: 新功能
- fix: 修复bug
- docs: 文档
- style: 代码风格（不影响功能）
- refactor: 重构
- test: 测试
- chore: 构建或依赖变更

**Scope** (可选):
- auth: 认证相关
- guide: 地陪相关
- order: 订单相关
- admin: 管理后台
- upload: 文件上传
- ui: UI组件

**Subject** (必须):
- 使用祈使句
- 不要大写首字母
- 不要以句号结尾
- 限制在50个字符以内

**Body** (可选):
- 详细说明改动内容
- 每行不超过72个字符

**Footer** (可选):
- 关闭相关Issue: Closes #123

### 示例

```
feat(auth): 实现用户注册功能

- 创建POST /auth/register API
- 实现密码加密和验证
- 添加手机号唯一性检查
- 生成JWT Token
- 编写单元测试

Closes #101
```

<type>/<feature-name>
